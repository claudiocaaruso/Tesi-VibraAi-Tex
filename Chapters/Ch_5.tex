\section{Exploratory PCA of the dataset}
\label{sec:pca_dataset}

In this work, PCA was first used as an exploratory tool to inspect the structure of the Raman dataset before building the classification model. This analysis should be interpreted as descriptive. PCA is unsupervised: it does not use
the tissue labels to determine the components, and therefore it is not expected to find
directions that are optimal for classification. Nevertheless, it is useful before
training a predictive model because it shows whether the spectral dataset contains
dominant variance patterns and how many components may be needed to obtain a compact representation of the
spectra.

\subsection{Explained variance of the principal components}
\label{subsec:pca_explained_variance_vibra}

The explained variance was first inspected to evaluate how the information in
\(\mathbf{X}\) is distributed across the principal components. The scree plot in
\Cref{fig:pca_scree_plot} shows that the first components explain most of the variance,
while the contribution of later components rapidly becomes smaller.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.65\textwidth]{Images/images_ch_3/07_scree_plot_explained_variance.png}
    \caption{Explained variance associated with the first principal components.  NOTA(grafico poco leggibile?)}
    \label{fig:pca_scree_plot}
\end{figure}

The cumulative explained variance, shown in \Cref{fig:pca_cumulative_variance}, gives
the corresponding global view. The curve has a steep initial increase and then gradually
flattens, indicating that the first components retain a large part of the total variance,
whereas additional components provide progressively smaller contributions. In this
analysis, the \(95\%\) threshold is reached by retaining the first \(89\) principal
components; this value was used as a practical reference for comparing reduced PCA
representations.

\begin{figure}[H]
    \centering
    \includegraphics[width=0.65\textwidth]{Images/images_ch_3/06_cumulative_explained_variance.png}

    principal components.}

\end{figure}

\Cref{fig:pca_reconstruction_comparison} shows the effect of reconstructing the same
spectrum with a very small number of components and with the \(89\) components selected

\subsection{Projection of spectra in PCA space}
\label{subsec:pca_projection_vibra}

After inspecting the variance distribution, the spectra were visualized in the PCA space
using score plots. For readability, the plots were generated on a random subset of
\(10000\) spectra, with points coloured according to the associated tissue label. In
these plots, PC1, PC2 and PC483 denote the scores of each spectrum along the corresponding
principal components, ordered from the largest to the smallest explained variance.

\begin{figure}[H]
    \centering
    \subfloat[Projection onto PC1 and PC2.\label{fig:pca_score_pc1_pc2}]{
        \includegraphics[width=0.48\textwidth]{Images/images_ch_3/08_pca_score_plot_pc1_pc2.png}
    }
    \hfill
    \subfloat[Projection onto PC1 and PC483.\label{fig:pca_score_pc1_pc483}]{
        \includegraphics[width=0.48\textwidth]{Images/images_ch_3/09_pca_score_plot_pc1_pc483.png}
    }
    \caption{Score plots of Raman spectra in the PCA space.}
    \label{fig:pca_score_plots}
\end{figure}

\Cref{fig:pca_score_pc1_pc2} shows that the data cloud is distributed along both axes,
although its spatial extension is larger along PC1. This is consistent with the larger
variance associated with the first principal component, while PC2 still describes a
visible part of the variability of the dataset. The coloured labels remain largely
overlapped in this plane, indicating that the first two components do not isolate the
tissue classes by themselves.

\Cref{fig:pca_score_pc1_pc483} makes the contrast between a dominant component and a late
component more evident. The points preserve a wide spread along PC1, whereas their
coordinates along PC483 are compressed close to zero. This visual difference reflects the
much smaller variance explained by the last component: in this projection, the structure
of the point cloud is therefore mainly determined by PC1.


\section{Hyperparameter selection}
\label{sec:hyperparameter_selection}

The configuration adopted in the final classification experiments was selected through
the grid search described in ***NOTA:vengono introdotte le due modalità grid e single?***, which compared different model
sizes, numbers of retained PCA components and batch sizes.

The comparison of the tested configurations showed that the best trade-off was obtained
with the small architecture (S), \(50\) retained PCA components and a batch size of
\(8192\). This configuration achieved the most favourable behaviour according to the
evaluation metrics introduced in \Cref{sec:evaluation_metrics-ch4}, while keeping both
the model and the input representation relatively compact.

The comparison between architectures also showed that larger models did not provide a
clear advantage for the present dataset. Although some configurations with more
components or larger networks could still obtain good numerical scores, their training
curves showed less stable behaviour and a larger discrepancy between training and
validation performance. The small model was therefore preferred because it provided a
more regular learning behaviour with lower computational cost and reduced risk of
overfitting.

The maximum number of epochs was set to \textbf{[check value]}, with early stopping based
on validation AUC and learning-rate reduction on plateau, as described in
\Cref{sec:training-ch4}. The classification threshold used to convert the predicted
probability into a binary label was set to \(\tau=\textbf{[check value]}\).

\section{Configuration of the final experiments}
\label{sec:final_experiments_configuration}

The selected configuration in \Cref{tab:selected_hyperparameters} was used to train the
final models on the two binary classification problems defined in
\Cref{sec:binary_classification-ch4}. 

\begin{table}[H]
    \centering
    \begin{tabular}{|l|c|c|}
        \hline
        \textbf{Experiment} & \textbf{Positive class} & \textbf{Negative class} \\
        \hline
        Tumour vs stroma & Tumour & Tumour stroma \\
        \hline
        Tumour vs non-tumour & Tumour & Valid non-tumour classes \\
        \hline
    \end{tabular}
    \caption{Binary classification experiments evaluated with the selected ANN configuration.}
    \label{tab:final_binary_experiments}
\end{table}

The first experiment focuses on the distinction between tumour and tumour stroma. This is
a more specific binary problem, in which the negative class is restricted to a single
histopathological label. The second experiment evaluates a broader tumour detection
setting, where tumour spectra are compared against the valid non-tumour tissue classes
available in the dataset. This second task is expected to be more complex, because the negative class is not formed
by a single tissue type, but groups together spectra from different non-tumour tissues.

For both experiments, the same evaluation strategy was adopted.
